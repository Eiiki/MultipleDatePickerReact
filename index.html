What is MultipleDatePickerReact ?

MultipleDatePicker is a React component to show a simple calendar allowing user to select multiple dates. It's a fork of http://arca-computing.github.io/MultipleDatePicker/

Get it bower style

bower install multiple-date-picker-react --save

Get it npm style

npm install multiple-date-picker-react --save

Get it git style

git clone https://github.com/arca-computing/MultipleDatePickerReact.git

Time to use it

Include files

<script type="text/javascript" src="multipleDatePicker.min.js"></script>
<link rel="stylesheet" type="text/css" href="multiple-date-picker.css"/>

Don't forget to include MomentJS, React and ReactDom js files

Options

<MultipleDatePicker
    month={} //any type MomentJS can use in constructor (moment date, timestamp, UTC formated date ...)
    sundayFirstDay={true/false} //boolean, false default
    highlightDays={myHighlightDays} //details below
    weekDaysOff=[0, 6] //days never selectable in a week (0 = Sunday, 1 = Monday ... 6 = Saturday)
    dayClick={dayClickCallback} //details below
    dayHover={dayHoverCallback} //details below
    changeMonth={monthChangeCallback} //details below
    callbackContext={this} //context given to every callback
/>
month

Any type MomentJS can use in constructor, default will be current month

highlightDays

Here is the major difference with Angular version, now we use only one parameter to change days state in calendar (except for recurrent week days off with weekDaysOff).

day: MOMENT/TIMESTAMP, //any type MomentJS can use in constructor
css: STRING, //chained css
notSelectable: BOOLEAN, //default false, true so day will not be selectable (dayClick callback not called and select state will not change)
selected: BOOLEAN, //selected or not by default
title: STRING //html title property

A simple example

var myHighlightDays = [
{
    day: moment(1438560000000), //03 august 2015 is my birthday
    css: 'birthday', //a beautiful color for this special day
    notSelectable: true, //can't change this magic day state
    selected: true,
    title: 'My birthday today !!!'
},
    {
    day: moment(1438646400000), //04 august 2015
    notSelectable: true //everyone is off this day, hangover -_-
},
    {
    day: moment(xxx), //a default selected day
    selected: true
},
    {
    day: moment(yyy), //another default selected day
    selected: true
}
];

With this property you can handle any kind of day : events, days-off, pre-selected days ...

dayClick

Callback called when a day is clicked, params are the day (MomentJS object for easy manipulations) and the select state before clicking.

Returning true will change the day select state.

var dayClickCallback = function(momentDate, selectedBefore){
    //do whatever you want
    //return true will change the select state
    //return nothing or false will not change the select state
};
dayHover

Callback called when mouse enter a day, params are the day (MomentJS object for easy manipulations) and the select state.

var dayHoverCallback = function(momentDate, selectedBefore){
    //do whatever you want
};
changeMonth

Callback called when trying to change month, params are a day in the month (MomentJS object for easy manipulation) dans the increment.

Returning true will allow changing month.

var monthChangeCallback = function(momentDate, increment){
    //example, we don't allow navigation in the futur or back to more than 2 months
    //we can inline condition but for demo purpose we use dummy if

    var newMonth = momentDate.add(increment, 'MONTH');
    if(newMonth.isBefore(moment().subtract(2, 'MONTH'))){
        return false;
    }

    if(newMonth.isAfter(moment())){
        return false;
    }

    return true;
};
